# Data Warehouse & Analytics Project

## 👋 Overview

Welcome to my Data Warehouse & Analytics repository! This project showcases an end-to-end data solution — starting from raw source files and ending in business-ready insights. I built it as a hands-on portfolio piece to practice real-world data engineering and analytics workflows.

---

## 🎯 Project Goals

### Part 1: Building the Warehouse (Data Engineering)

**Goal**
Set up a modern SQL Server data warehouse that brings sales data from multiple systems into one place, ready for reporting and analysis.

**What this covers**
- **Sources:** Pull in data from two systems — ERP and CRM — delivered as CSV exports.
- **Cleaning:** Identify and fix data quality problems before anything gets analyzed.
- **Combining data:** Merge both sources into one clean, easy-to-query model.
- **Scope:** Only the most recent snapshot of data is needed — no need to track historical changes.
- **Docs:** Clearly document the final data model so both technical and non-technical users can understand it.

### Part 2: Turning Data into Insights (Analytics)

**Goal**
Write SQL queries that surface meaningful patterns around:
- How customers behave
- How products are performing
- How sales trends move over time

The end result gives stakeholders the numbers they need to make smarter decisions.
